本文件的目标不是要求我懂技术，而是让 AI 主动规划最适合我的技术方案。
我只做决策，不做技术判断。然后

──────────────────────────────────────────
🧱 1. 技术栈选择规则（核心更新）
──────────────────────────────────────────

AI 开始任何开发前必须先完成：

1. 根据项目需求给出 2–3 套技术栈方案
2. 每套必须包含：

   - 框架选择理由（如 Next / Svelte / React / Astro）
   - 开发难度（低/中/高）→ 用人类能懂的方式描述
   - 安全性与未来可维护性
   - 成本与风险

3. AI 必须用提问方式引导我选择  
    例：
   你更需要哪种方向？

A. 上线最快、开发简单（但扩展性一般）
B. 性能更强、SEO 友好（适合规模扩大）
C. 适合长期维护、多终端适配

──────────────────────────────────────────
🧱 2. 默认推荐（当我不做决定时）
──────────────────────────────────────────

- Next.js + TypeScript（SSR + SEO）
- TailwindCSS（快速开发）
- 部署 Vercel（最省心/对前端友好）

仅在我明确说「用默认」或我未回应时执行。

──────────────────────────────────────────
🤖 3. AI 在开发前必须执行的 Checklist
──────────────────────────────────────────

- 读取以下内容：
- spec.md（核心需求）
- style.md（UI 规范）
- content.md（页面内容）
- rules.md（沟通方式）
- 用通俗语言向我描述方案，让我听得懂
- 不得跳过询问阶段就开始写代码

──────────────────────────────────────────
🔐 4. 安全要求（不可忽视）
──────────────────────────────────────────

- API Key 必须在后台、不可暴露前端
- 所有密钥都需使用 .env 管理
- 防止滥用：AI 应给我节流/限制机制建议
- 部署前必须提醒我检查安全配置

──────────────────────────────────────────
📁 5. 项目结构（简单且一致）
──────────────────────────────────────────
src/app/ → 页面与路由
src/components → 可复用组件
src/lib/ → 工具函数/AI 接口
src/styles/ → 全局样式（来自 design-style.md）

如 AI 认为需要改变结构 → 必须先解释好处再修改。

──────────────────────────────────────────
🚀 6. 开发与交付要求
──────────────────────────────────────────

- 输出代码必须能直接运行，不留 TODO 片段
- 每完成一个模块都需要我确认
- 最终交付内容必须包含：
- 运行方式（人能看懂）
- 部署方式（一步步指导）
- 环境变量说明
